원격 시동(WOL) 및 VPN 접속을 위한 Iptime 공유기 설정

  1. 보안 및 네트워크 관련
    • iptime 제조사의 서비스 페이지 에 가면 링크된 포스트와 같이 다양한 가이드를 안내해준다.
    • 우선 관리자 계정을 생성한 뒤, 사용하고 있는 Iptime의 펌웨어를 최신으로 업데이트 해준다.
    • 국가별 접속제한 기능 등 iptime의 여러 보안 기능을 설정한다.
  2. vpn 관련
  • Iptime에 WireGuard VPN을 이용하여 내부망으로 접근하게 하려면 WireGuard 서버 설정을 실행한 다.
  • 자신의 컴퓨터, 핸드폰 등에 WireGuard VPN 앱을 설치한 뒤 공개키를 입력하여 등록하면 된다.

  1. WOL 설정

    Iptime 에서 설정해야 할 부분과 Proxmox 서버에서 설정해야할 부분이 나뉜다.

    • Proxmox 서버

      • ethtool 설치 apt intall ethtool (이미 설치되어 있을 거임)

      • 현재 사용 어댑터 명, Mac, IP주소 확인 ip addr

        • 차례대로 emp1s0, a8:a1:59:5a:ba:c2, 192.168.1.200/24

      • Wake-on 확인 ethtool [어댑터이름] d면 현재 꺼져있는 상태

      • WOl활성화 ethtool -s [어댑터이름] wol g

        • ethtool -s enp1s0 wol g

      • 인터페이스 설정으로 가서 재부팅시 자동 wol 옵션 추가

        • vi /etc/network/interfaces
        • post-up /usr/sbin/ethtool -s [인터페이스이름] wol g 를 추가
        • post-up /usr/sbin/ethtool -s enp1s0 wol g
    auto lo
iface lo inet loopback

iface enp1s0 inet manual
post-up /usr/sbin/ethtool -s enp1s0 wol g
auto vmbr0
iface vmbr0 inet static
        address 192.168.0.21/24
        gateway 192.168.0.1
        bridge-ports enp1s0
        bridge-stp off
        bridge-fd 0

iface wlp3s0 inet manual
    • 이후 Iptime 에서 WOL 메뉴로 들어가 PC 이름과 MAC 주소를 등록하면 된다.

로컬피씨에서 vm 으로 만든 우분투 ssh접속 하는법

  1. SSH 키 페어 생성 (로컬 PC에서)

먼저 SSH 접속에 사용할 키 페어(공개 키와 개인 키)가 없다면 생성해야 한다.

ssh-keygen -t ed25519

  • **ed25519**는 rsa 등 다른 암호화 알고리즘보다 더 현대적이고 안전한 방식으로 권장된다고 한다.
  • 키를 저장할 위치를 물으면 기본 경로(예: ~/.ssh/id_ed25519)를 그대로 사용하기 위해 Enter 키를 누른다.
  • 암호(passphrase)를 입력하라는 메시지가 나타나면, 암호 없이 편리하게 접속하고 싶을 경우 그냥 Enter 키를 두 번 누른다. (나 같은 경우는 간단히 vm이라고 따로 입력하였다.)
  1. 공개 키 내용 복사 (로컬 PC에서)

생성된 공개 키의 내용을 복사한다

cat ~/.ssh/id_ed25519.pub

  • **ssh-ed25519**로 시작해서 사용자 이름과 호스트명으로 끝나는 전체 문자열을 복사한다.
  1. Proxmox GUI에서 공개 키 등록

이제 복사한 공개 키를 Proxmox 웹 GUI를 통해 VM에 등록한다.

  1. Proxmox 웹 GUI에 접속하여 해당 Ubuntu VM을 선택

  2. Cloud-Init → SSH Public Key 입력란에 위에서 복사한 공개 키 전체를 붙여 넣는다

  3. User 항목에 VM에 생성할 사용자 이름을 지정한다. Ubuntu 클라우드 이미지의 기본 사용자는 보통 ubuntu 이지만 나는 따로 설정하였다.

  4. Hardware 탭에서 CloudInit Drive가 추가되어 있는지 확인한다. 만약 없다면 **Add > CloudInit Drive**를 통해 추가해야 한다. Proxmox는 이 가상 CD-ROM 드라이브를 통해 VM에 설정 정보를 전달한다.

  5. VM 재부팅 및 SSH 접속

설정을 적용하기 위해 VM을 재부팅 후, 로컬 PC의 터미널에서 다시 SSH 접속을 테스트를 한다. 이때 root가 아닌 Cloud-Init에서 설정한 사용자 이름(예: ubuntu)으로 접속해야 한다. 예를 들자면

ssh ubuntu@192.168.0.59 이다.

참고: “Host key” 경고 메시지에 대하여**

 The authenticity of host... can't be established. 메시지는 VM 서버 IP 주소에 처음 접속하기 때문에 나타나는 자연스러운 경고다. 하지만 메시지 내용 중 **This host key is known by... 192.168.0.60**라는 부분은, 과거에 192.168.0.60이라는 다른 서버에 접속했던 기록이 있는데, 지금 접속하려는 192.168.0.59 서버가 동일한 호스트 키를 사용하려고 할 때 발생할 수 있다. 이는 VM을 복제했거나 IP를 재사용한 경우 발생한다.

 **yes**를 입력하여 키 정보를 갱신하면 Permission denied 오류만 해결하면 된다. 로컬 PC의 ~/.ssh/known_hosts 파일에서 해당 IP 주소와 관련된 줄을 삭제하거나 다음 명령어로 간단히 삭제할 수 있다. ssh-keygen -R 192.168.0.59

마무리

이로써 기본적인 구성은 완료하였고, 추후 노드를 추가 증설하거나 스펙업을 하게 될 시 Proxmox backup server, HA (high availability) 기능을 더 구축하려고 한다. 이 외에도 VLAN Zone을 구축하고 싶은데 아무래도 네트워크 및 보안 관련 지식이 없다 보니 시간 날 때 공부하면서 설정하려고 한다.
