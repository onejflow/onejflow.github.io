logging 을 이용한 로그관리

Python logging 모듈 주요 구성요소

  1. class logging.Logger
    • Logger.setLevel(), Logger.addHandler(), Logger.addFilter() 등을 사용할 수 있게 한다.
    • Logger는 Singleton 패턴 이다. 즉, getLogger를 한다고 매번 새로운 instance가 return되는게 아니라 이미 그 이름으로 존재하는 Logger를 return 한다.
    • 그리고 Observer 패턴 을 동시에 따르며, 로그 레코드가 생성될 때 로그 핸들러(Observer) 객체에 자동으로 알림(event catch)을 보냄. 로그 레코드 생성자는 로그 레코드가 생성되면 모든 등록된 옵저버(핸들러)에게 보낸다.
    • 메시지를 생성하는 시점에 “thread-safe” 하게 처리된다.
    logger = logging.getLogger("로거이름")
  2. class logging.Filter
    • 출력될 로그를 필터링한다. 별도 클래스로 필터링 로직을 작성하고 이미 생성한 로거나 핸들러에 addFilter() 메서드로 추가한다.
    class CustomFilter(logging.Filter):
    def filter(self, record):
        return 'skip-this' not in record.msg
  3. class logging.Formatter
    • 출력될 로그의 포맷을 설정하며 다양한 포맷이 있으므로 공식문서를 참고하는게 좋다.
     formatter = logging.Formatter(
    '%(asctime)s - %(name)s - %(levelname)s - %(message)s'
    )
# 결과 예시 
# 2025-07-05 12:57:27,599 - src.trade_executor - INFO - This is message
  4. class logging.Handler
    • 로그 메세지를 처리할 방식을 설정한다.
    • 주로 StreamHandler(콘솔창에 로그 메세지 출력)와 FileHandler(로그 메세지 저장)를 사용한다.
      • FileHandler의 경우 지정된 단일 파일에 계속 추가하여 로그 파일의 크기를 제한하지 않기 때문에 주의가 필요하며 이를 위해 RotatingFileHandler를 사용한다.
      • RotatingFileHandler는 로그 파일의 크기를 관리하고, 오래된 로그 파일을 일정 개수만큼만 유지하여 디스크 공간을 효율적으로 사용하게 해준다.
    # 콘솔창에 출력하는 핸들러 설정
console_handler = logging.StreamHandler()
# my_app.log 파일에 로그 기록
file_handler = logging.FileHandler('my_app.log') 
# maxBytes=1024*1024 (1MB), backupCount=3 (최대 3개의 백업 파일 유지)
rotate_handler = RotatingFileHandler('my_app_rotating.log', maxBytes=1024*1024, backupCount=3)

로그레벨

  • DEBUG
    • 개발/테스트 단계에서 각 기능이 올바르게 작동하는지 확인할 때 사용하는 수준, 실제 운영 중에 남기지 않을 로그
  • INFO
    • 실제 운영 중인 프로그램을 실행할 때 발생하는 정상적인 이벤트 알림
    • 각 기능의 시작, 종료 시점, 반드시 알아야하는 정보들을 INFO 수준으로
  • WARNING
    • 현재는 괜찮지만 잠재적으로 문제가 될 수 있는 작동일 때 남기는 로그 수준
    • 예를 들면 ML 모델의 예측값이 상정했던 범위 밖
    • WARN 수준의 로그는 기록되더라도 프로그램의 실행이 중단되지 않음
  • ERROR
    • 프로그램이 정상 작동하지 않았을 경우 사용
    • 심각한 오류나 예외 상황에 대해서만 사용 -> 즉각적인 조치 필요
    • WARN 수준과 확실한 구분이 필요함
  • CRITICAL
    • 심각한 오류이며 프로그램 자체가 계속 실행되지 않을 수 있을 때

Python의 logging 모듈의 기본 로깅 수준은 WARNING 이다. 그렇기에 이 수준 또는 그보다 심각한 수준의 일은 자동으로 로그로 남는다. basicConfig 메서드로 기본 로깅 수준 변경이 가능하다.

Python 로그 예시

로깅 모듈 작성 및 설정하는 logging_config.py

import logging
from logging.handlers import RotatingFileHandler

log_format = '%(asctime)s - %(name)s - %(levelname)s - %(message)s'
_loggers = {}  # 로거 캐시

def get_stream_handler():
    """Stream 핸들러"""
    console_handler = logging.StreamHandler()
    console_handler.setLevel(logging.DEBUG)
    console_handler.setFormatter(logging.Formatter(log_format))
    return console_handler

def get_file_handler(log_file='application.log'):
    """파일(Rotating) 핸들러"""
    file_handler = RotatingFileHandler(
            log_file, 
            maxBytes=1024*1024, # 1MB
            backupCount=5,
            encoding='utf-8')
    file_handler.setLevel(logging.INFO)
    file_handler.setFormatter(logging.Formatter(log_format))
    return file_handler

def get_logger(name):
    """로거 생성 (중복 방지)"""
    if name in _loggers:
        return _loggers[name]
    
    logger = logging.getLogger(name)
    
    # 핸들러가 이미 있는지 확인
    if not logger.handlers:
        logger.setLevel(logging.DEBUG)
        logger.addHandler(get_stream_handler())
        logger.addHandler(get_file_handler())
        logger.propagate = False  # 루트 로거로 전파 방지
    
    _loggers[name] = logger
    return logger
    
if __name__ == "__main__":
    # 로깅 설정 테스트
    logger = get_logger(__name__)
    
    class CustomFilter(logging.Filter):
        def filter(self, record):
            return 'data-check' not in record.msg
    logger.addFilter(CustomFilter())
    
    logger.debug("이것은 디버그 메시지입니다.")
    logger.info("이것은 정보 메시지입니다.")
    logger.info("['data-check'] 이것은 정보 메시지입니다.")
    logger.warning("이것은 경고 메시지입니다.")
    try:
        1 / 0
    except ZeroDivisionError:
        logger.error("이것은 오류 메시지입니다.")
        logger.exception("예외가 발생했습니다!")

logging_config를 사용하는 main.py

import logging
from logging_config import get_logger

# 모듈 레벨에서 로거 생성
logger = get_logger(__name__)

def main():
    try:
        logger.info("Program starts")
        # 애플리케이션 로직
        logger.info("Program finished successfully")
    except Exception as e:
        logger.exception("Program failed: %s", e)

if __name__ == "__main__":
    main()

# 결과 예시
# 2025-07-05 12:57:27,600 - __main__ - INFO - Program starts 라는 로그가 콘솔에 출력 및 application.log 란 로그파일에 저장된다

loguru

logging이 파이썬 기본 내장 라이브러리라면 loguru는 외부 라이브러리 이며 logging모듈에 의존하지 않고 from scratch 부터 새로 작성한 로깅 라이브러리이고 좀 더 간단하게 사용가능하다.

위에서 보다시피 logging모듈을 사용하려면 logging을 위한 여러 설정 및 기능을 알아야한다. 세세한 설정을 하기엔 logging모듈이 좋지만, 그럴필요 없을 경우 여러 handler를 설정하거나 옵션을 주는게 피곤할 수 있다.
예를 들면 logging 같은 경우

  • handler 인스턴스를 설정하고
  • logger에 addHandler메서드를 통해 추가를 해줘야 한다. luguru같은 경우
  • add 메서드 한번에 가능하다

그 외에도

  • @logger.catch 데코레이터로 예외 자동처리를 하여 디버깅 편리
  • JSON포맷, 비동기 로깅, 다중 로그 대상 등 확장성 관련도 편리

위 logging_config.py를 loguru로 변경한다면 다음과 같이 설정할 수 있다.

# loguru_config.py
import sys
from pathlib import Path

from loguru import logger

log_format = "{time:YYYY-MM-DD HH:mm:ss} | {name} | {level} | {message}"
_LOGGERS = set()  # 캐시: 불필요한 add 제거용
LOG_PATH = Path("log/loguru_application.log")


def _filter(record):
    return "data-check" not in record["message"]

def _configure_once():
    if "configured" in _LOGGERS:
        return

    logger.remove()  # 기본 콘솔 제거
    # 터미널에 출력을 위한 설정 - logging에 stream 핸들러와 유사
    logger.add(sink=sys.stdout, level="DEBUG", filter=_filter)
    # 로그 파일 저장을 위한 설정 - file 핸들러와 유사
    logger.add(
        sink=LOG_PATH,  # log 저장 경로
        level="INFO",
        rotation="1 MB",  # 1mb 단위 분할
        retention=5,  # backup을 5개까지 보관
        encoding="utf-8",
        format=log_format,
        enqueue=True,  # 멀티프로세스 안전
        filter=_filter,
    )

    _LOGGERS.add("configured")


def get_logger_(name: str = None):
    """
    loguru는 logger 객체가 싱글턴이지만, .bind(name=…) 를 통해
    '이름' 태그를 붙이면 모듈별 구분(logging.getLogger(name)과 유사)
    """
    _configure_once()
    # 동일 name으로 bind 중복 생성 방지
    if name in _LOGGERS:
        return logger.bind(name=name)
    _LOGGERS.add(name)
    return logger.bind(name=name)


# 테스트용 실행
if __name__ == "__main__":
    log = get_logger_(__name__)

    log.debug("#" * 10 + "프로그램 시작" + "#" * 10)
    log.debug("디버그 메시지")
    log.info("정보 메시지")
    log.info("['data-check'] 정보 메시지")
    log.warning("경고 메시지")

    @logger.catch(level="ERROR", message="예외 발생")
    def divide():
        return 1 / 0

    divide()
    log.debug("#" * 10 + "프로그램 종료" + "#" * 10)
