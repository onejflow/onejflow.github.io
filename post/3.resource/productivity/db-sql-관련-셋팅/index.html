디비버

  1. 쿼리 자동 대문자 전환
    • 윈도우 > 설정 > 편집기 > SQL 편집기 > SQL포맷 → keyword case Upper
  2. 라인번호 표기
    • 윈도우 > 설정 > 편집기 > 문서편집기 → 행 번호 표시
  3. 인코딩 설정
    • 윈도우 > 설정 > 일반 > 작업공간 → 텍스트파일인코딩 기타 UTF-8
  4. 자동 저장
    • 윈도우 > 설정 > Userinterface > 편집기 > Autosave → 하단 interval에서 초단위로 자동저장
  5. 괄호, 따옴표 자동 완성
    • 윈도우 > 설정 > 편집기 > SQL 편집기 > Code Editor → Auto close 체크
  6. 쿼리 작성 중 테이블 자동완성 목록에서 클릭하면 테이블 옆 Space와 Alias 자동 생성 막는 방법
    • 편집기 - SQL 편집기 - SQL 자동완성 으로 이동한다.
    • Insert space after table/column names 는 체크를 해제한다. (자동완성 중 테이블 및 컬럼을 클릭하면 자동으로 Space가 들어가는 기능)
    • Insert table aliases (in FROM clause): 는 N/A로 변경한다. (자동완성 중 테이블을 클릭하면 자동으로 Alias가 생성되는 기능)
  7. 테이블 코멘트, 컬럼코멘트 표시하기
    • 테이블
      • 윈도우 > 설정 > User Interface > Navigator    : Navigator > Show Objects description
    • 컬럼
      • 윈도우 > 설정 > Editor > data editor > appearance > common > show column description in header

가독성을 위한 SQL 스타일 가이드

SQLFluff

SQLFluff는 dbt와 같은 templating tools에 사용가능한 SQL Linter 이며 보다 나은 코딩스타일을 유지할 수 있음

  1. 예약어는 대문자로 작성

    • SELECT, FROM, WHERE 등

  2. field 이름은 snake_case

    • id 값 같은경우 데이터 결합시 모호성 방지위해 필드 앞에 데이터 소스 붙이기
      • ex) table1.account_id AS table1_account_id
    • Boolean인 경우 has_, is_, does_ 로 구분
    • Timestamp 는 _at 으로 끝나며 언제나 UTC기준
    • date는 _date으로 끝내기

  3. Alias로 별칭을 지을 때는 명시적인 이름 사용

    • AS a, AS b 가 아닌 명시적인 것을 사용 JOIN 할 때 테이블 또한 마찬가지

  4. 예약어 기준 줄바꿈 및 왼쪽 정렬 기준 작성

    • 컬럼 구분시 쉼표는 뒤로
    WITH temp AS (
  SELECT
    first_col,
    second_col
  FROM 
    Table
  WHERE
    second_col = "example"
)

  5. GROUP BY, ORDER BY 등 무엇을 기준으로 했는지 명시적으로 작성

  6. 들여쓰기

    • 쿼리 내의 들여쓰기는 Tab 대신 space로 구분하며 2칸으로 들여쓰기.
    • CTE 내에서 전체 SQL 문은 4칸 들여쓰기
